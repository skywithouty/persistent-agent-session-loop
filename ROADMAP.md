# Persistent Session Loop Roadmap

## Phase 1: File Board and Claude Loop MVP

Goal: prove that one long-running Claude Code session can poll and execute a Codex task through local files.

Deliverables:

- `.session-loop/` directory scaffold
- `STATE.json`
- one example Codex task
- Claude `/loop` prompt
- one Claude report
- one Codex review

Success criteria:

- no `claude -p`
- no stateless batch loop
- Claude keeps the same Cursor terminal session
- Codex can understand the report and decide the next step

Estimated time: 0.5 to 1 day.

## Phase 2: Codex App Review Rhythm

Goal: make Codex App act as the same-thread planner/reviewer for repeated rounds.

Deliverables:

- Codex review checklist
- task creation checklist
- state transition checklist
- concise memory update format
- manual user decision gate rules

Success criteria:

- Codex asks the user only at meaningful gates
- Codex issues small tasks
- Codex reviews actual diffs and reported verification

Estimated time: 0.5 to 1 day.

## Phase 3: Claude Statusline Context Monitor (deprecated)

**Deprecated (round 17):** The custom context monitoring infrastructure delivered in this phase is deprecated. Claude Code's default auto-compact replaces it. Scripts (`claude-statusline.ps1`, `check-compact-gate.ps1`) remain as legacy files but are not required for active loop operation. Context threshold config (`context_compact_threshold`) has been removed from `STATE.json`. The `## Context Usage` report section is retained as an optional stub.

Original goal: expose Claude Code context usage to local files.

Original deliverables:

- statusline script (exists: `.session-loop/context/claude-statusline.ps1`)
- `.session-loop/context/claude-status.json` (generated by the script)
- context threshold config (removed from `STATE.json`)
- report section for context usage (optional stub)

Original estimated time: 0.5 to 1 day.

## Phase 4: Compact Bridge (deprecated)

**Deprecated / Cancelled (round 16–17):** All custom compact infrastructure described below is deprecated. The project relies exclusively on Claude Code's default auto-compact. Scripts (`claude-statusline.ps1`, `check-compact-gate.ps1`) remain as legacy files but are not required, not installed into new projects, and not validated by doctor. Bridge process development (Phase 4b/c) is cancelled.

**Phase 4a (historical):** Post-Task Compact Gate was implemented and live-validated in round 12, then deprecated by user decision in round 16. The custom 60% threshold, `check-compact-gate.ps1`, and `context_at_threshold` blocking are no longer active.

**Phase 4b/c (cancelled):** Bridge process and automatic `/compact` injection will not be built. Claude Code auto-compact replaces the custom compact infrastructure.

Original goal: reliably compact the Claude session around 60% context usage.

Historical deliverables (Phase 4a):

- Post-Task Compact Gate in `/loop` and report routines
- Statusline script with live/mock/unknown/error source distinction
- Compact policy docs (PHASE4_COMPACT_BRIDGE_DESIGN.md)
- `check-compact-gate.ps1` — deterministic PowerShell checker with `-Apply` switch
- Live validation: checker blocked at 66% → manual `/compact` → checker verified 18% → `-Apply` restored `next_actor = "codex"`

Cancelled deliverables (Phase 4b/c):

- Bridge process that launches Claude Code
- Automatic `/compact` injection
- Compact summary prompt and bridge logs

Original estimated time: 2 to 4 days.

## Phase 5: Cross-Project Installer

Goal: install the persistent session loop into any new research or paper-reproduction project with one command.

Deliverables:

- `PHASE5_INSTALLER_DESIGN.md` -- design spec (complete)
- `doctor.ps1` -- read-only installation validator (complete, @() fix round 23)
- installer script (`install.ps1`) -- install / -Check / -WhatIf modes (complete)
- uninstall or cleanup command (`uninstall.ps1`) -- safe removal with hash matching (complete, removedCount fix round 23)
- template files (complete)
- README with copy-paste commands (complete)
- full Phase 5 fixture sweep (complete round 23, 11/11 PASS)
- GitHub-hosted install path (complete — published to https://github.com/skywithouty/persistent-agent-session-loop)

Success criteria:

- user can initialize a new project from GitHub (complete)
- no manual file copying is required
- installed files are inspectable and project-local
- existing project files are not overwritten without confirmation

Estimated time: 1 to 2 days after Phase 4.

## Phase 6: Research Workflow Hardening

Goal: make the loop useful for paper reproduction work.

Deliverables:

- natural-language cross-project quick start (round 26 — README and START docs now include copy-paste prompts)
- Codex reviewer heartbeat docs (round 27 — two-loop explanation, heartbeat rule in CODEX_REVIEW_RHYTHM.md)
- paper reproduction goal template
- experiment report template
- dataset and model download gates
- command log summary
- metrics comparison report
- final reproduction report template

Success criteria:

- the system can guide a paper reproduction from setup through baseline run
- costly or risky steps ask the user
- all results are traceable through files

Estimated time: 2 to 5 days depending on paper complexity.

## Recommended Next Step

Phase 3 and Phase 4 custom context infrastructure is deprecated. The project relies on Claude Code default auto-compact.

Phase 5 is complete. The project is published to GitHub at `https://github.com/skywithouty/persistent-agent-session-loop`. Install, doctor/check, uninstall, fixture sweep (11/11 PASS), and cross-project E2E validation are all done. The repository excludes `.session-loop/` and `.claude/` runtime history; installed projects generate their own local task board state.

The next engineering phase is **Phase 6: Research Workflow Hardening** — making the loop useful for paper reproduction work with experiment templates, dataset/model download gates, command log summaries, and metrics comparison reports.
